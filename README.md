Описание проекта:
2d top down игра,  в которойпредстоит играть за ниндзю. В игре есть 8 уровней сложности и 9 типов врагов.

Архитектура и подходы
Компонентная архитектура: логика разнесена по компонентам (PlayerMover, PlayerAnimator, PlayerAttacker, CollisionHandler, PlayerSounds), основной класс Player координирует их работу.

Интерфейсы и абстракции:
IInputReader — абстракция ввода (направление, атака, ускорение, взаимодействие), что даёт смену способа ввода (клавиатура/геймпад) без изменения логики игрока.
IInteractable — единый контракт для всех взаимодействуемых объектов (двери, сундуки и т.п.) с поддержкой замков и ключей.
IItem — общий интерфейс для собираемых предметов.
События (events): уведомления о смерти, эффектах удара/смерти/ускорения; подписка на события коллизий (взаимодействия, аптечки, ключи) и инвентаря.

Коллизии обрабатываются единым CollisionHandler с определением типов через TryGetComponent и разведением по событиям.

Базовая логика здоровья вынесена в абстрактный Character с классом Health и привязкой HealthBar, переопределение OnTakingDamage/OnDied.

Реализована конфигурируемая стейт-машина для врагов: состояния задаются массивом в инспекторе, у каждого состояния — список действий (StateMachineAction) и переходов (StateMachineTransition). Действия выполняются каждый кадр через абстрактный Act(); переходы решаются компонентами StateMachineDecision (Decide() → true/false) с выбором следующего состояния по строковому ID. Логика поведения вынесена в переиспользуемые Action и Decision.

Система настроек звука и музыки с помощью toggle и slider

Игра реализована под Desktop и Android
